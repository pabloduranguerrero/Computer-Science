Temario NM · A1.1

A1.1 Hardware y funcionamiento de las computadoras

Pregunta guía

¿Cómo cooperan CPU, memoria y dispositivos para ejecutar programas?

Introducción

Este subtema estudia la arquitectura física básica de un ordenador para entender cómo se ejecutan programas en la práctica.

Explicación detallada

La CPU ejecuta instrucciones mediante un ciclo continuo: búsqueda, decodificación y ejecución. La unidad de control coordina señales; la ALU realiza operaciones aritméticas y lógicas; y los registros almacenan datos temporales de alta velocidad.

La RAM mantiene programas y datos activos, mientras que el almacenamiento secundario (SSD/HDD) conserva información de forma persistente. Los dispositivos de entrada (teclado, ratón, sensores) aportan datos y los de salida (monitor, altavoces, impresora) muestran resultados. El rendimiento real depende del equilibrio entre estos componentes.

Conceptos clave

  • CPU, ALU y unidad de control
  • RAM (volátil) vs almacenamiento (no volátil)
  • Dispositivos de entrada y salida
  • Arquitectura básica del ordenador

Ejemplo práctico

Al abrir navegador, editor y videollamada, la CPU alterna procesos, la RAM sostiene los datos activos y el SSD carga recursos. Si falta RAM, el sistema se vuelve más lento aunque la CPU sea buena.

Aplicaciones en el mundo real

Selección de equipos para aula, dimensionamiento de laboratorios, optimización de rendimiento y diagnóstico técnico.

Errores frecuentes de los estudiantes

Confundir RAM con almacenamiento permanente y asumir que solo la CPU determina la velocidad del sistema.

Conexiones con otros temas del curso

A1.3 (el SO gestiona CPU/RAM), B2 (programas eficientes), A4.1 (requisitos de hardware en ML).

Resumen

  • El ordenador funciona por cooperación de CPU, memoria, almacenamiento y E/S.
  • La RAM acelera tareas activas; el almacenamiento conserva datos.
  • Comprender arquitectura ayuda a tomar decisiones técnicas reales.
