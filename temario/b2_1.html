B2.1 Fundamentos de programación

Pregunta guía

¿Cómo se construyen los programas informáticos a partir de instrucciones, datos y estructuras básicas que permiten resolver problemas de forma lógica y automática?

Introducción

La programación es una de las bases de la informática. Gracias a ella, los ordenadores, los móviles, las páginas web y las aplicaciones pueden ejecutar tareas, procesar información y responder a las acciones de las personas usuarias.

Sin embargo, programar no consiste simplemente en escribir líneas de código. Programar significa dar instrucciones precisas a una máquina para que realice una tarea concreta. Para ello, es necesario entender cómo se organiza la información, cómo se representa en un programa y cómo se pueden construir soluciones paso a paso.

En este tema se estudian los fundamentos de programación, es decir, los elementos más básicos y esenciales que permiten empezar a crear programas. Estos fundamentos sirven como base para todo lo que vendrá después: estructuras de control, algoritmos, funciones, programación orientada a objetos y desarrollo de aplicaciones más complejas.

Cuando una persona empieza a programar, suele pensar que lo más importante es aprender muchos comandos o mucha sintaxis. Sin embargo, lo más importante al principio es entender bien las ideas fundamentales:

  • qué es un programa
  • qué es una instrucción
  • cómo se almacenan datos
  • qué son las variables
  • qué tipos de datos existen
  • cómo entra y sale la información de un programa

Estas ideas son esenciales porque están presentes en prácticamente cualquier lenguaje de programación. Aunque la sintaxis de Python, Java o JavaScript sea distinta, los conceptos básicos son muy similares.

En este apartado nos centraremos en:

  • qué es un programa
  • variables
  • tipos de datos
  • entrada y salida

El objetivo es que el alumnado comprenda no solo cómo escribir programas simples, sino también qué está ocurriendo realmente cuando un programa se ejecuta.

Explicación detallada

¿Qué es un programa?

Un programa es un conjunto de instrucciones escritas en un lenguaje de programación que indican a un ordenador qué debe hacer. Estas instrucciones se ejecutan en un orden determinado y permiten resolver una tarea concreta. Un programa puede ser muy sencillo, como mostrar un mensaje en pantalla, o muy complejo, como una red social, un videojuego o un sistema bancario.

Una idea fundamental es que el ordenador no entiende intenciones humanas generales. Necesita instrucciones claras, exactas y ordenadas. Si queremos que un programa calcule la suma de dos números, debemos indicarle que lea el primer número, lea el segundo, los sume y muestre el resultado.

Por ejemplo, este programa muy simple muestra un mensaje:

print("Hola")

Aunque sea muy sencillo, ya cumple la definición de programa, porque es una instrucción escrita en un lenguaje que el ordenador puede ejecutar.

Es importante no confundir programa con algoritmo. Un algoritmo es la secuencia lógica de pasos para resolver un problema. Un programa es la implementación de ese algoritmo en un lenguaje que la máquina puede ejecutar.

Instrucciones y secuencia de ejecución

Un programa está formado por instrucciones. Una instrucción es una orden concreta que el ordenador debe ejecutar. Algunas instrucciones sirven para guardar datos, otras para hacer cálculos, otras para mostrar resultados y otras para controlar el flujo del programa.

Por ejemplo:

x = 10
y = 20
resultado = x + y
print(resultado)

En este caso, el programa ejecuta las instrucciones en orden: primero guarda 10 en x, después 20 en y, luego calcula la suma y finalmente muestra el resultado. Este orden es importante, porque si las instrucciones cambian de posición, el comportamiento del programa también puede cambiar.

Variables

Una variable es un espacio de memoria con un nombre que se utiliza para almacenar un dato que puede cambiar durante la ejecución del programa. Se puede imaginar como una caja con etiqueta: la etiqueta es el nombre de la variable y el contenido es el valor almacenado.

Por ejemplo:

edad = 16

Aquí, edad es el nombre de la variable y 16 es el valor que contiene.

Se llaman variables porque su valor puede variar. Por ejemplo:

puntos = 10
puntos = 15

Primero la variable puntos vale 10 y después pasa a valer 15.

Cuando escribimos una instrucción como x = 7, estamos haciendo una asignación. Esto no significa “x es igual a 7” en sentido matemático, sino “guardar el valor 7 en la variable x”.

También puede producirse una reasignación:

contador = 1
contador = contador + 1

Aquí, el nuevo valor de contador pasa a ser el valor anterior más 1.

Elegir buenos nombres para las variables es muy importante. Por ejemplo, un programa se entiende mejor si usa nombres como nota_media o precio_final en lugar de nombres poco claros como x o dato1.

Tipos de datos

Los tipos de datos indican qué clase de información se almacena en una variable y qué operaciones pueden realizarse con ella. No todos los datos son iguales: un número entero, una palabra o un valor verdadero/falso se comportan de manera distinta.

Los tipos básicos más habituales son:

  • Enteros: números sin decimales, como 5, -2 o 0.
  • Reales o decimales: números con parte decimal, como 3.5 o 19.99.
  • Cadenas de texto: secuencias de caracteres, como "Hola" o "Ana".
  • Booleanos: valores lógicos, que solo pueden ser True o False.

Ejemplos:

edad = 17
altura = 1.72
nombre = "Carlos"
aprobado = True

Es fundamental entender que 5 no es lo mismo que "5". El primero es un número y el segundo es texto. Esta diferencia afecta a las operaciones que se pueden hacer con cada dato.

A veces es necesario convertir datos de un tipo a otro. Por ejemplo, los datos introducidos por teclado suelen llegar como texto, por lo que si queremos operar con ellos matemáticamente tendremos que convertirlos:

edad_texto = input("Introduce tu edad: ")
edad_numero = int(edad_texto)

Entrada de datos

La entrada es la información que recibe un programa para poder trabajar con ella. Esa entrada puede proceder del teclado, de un archivo, de sensores, de una base de datos o de Internet. En los programas básicos, lo más habitual es obtenerla del teclado.

Ejemplo:

nombre = input("Introduce tu nombre: ")

Aquí el programa pide al usuario que escriba su nombre y lo guarda en una variable.

La entrada es importante porque hace que el programa sea interactivo. Si un programa no pudiera recibir datos, solo podría realizar siempre las mismas acciones.

Salida de datos

La salida es la información que el programa muestra, devuelve o produce como resultado. En los programas básicos suele mostrarse por pantalla, aunque también puede enviarse a archivos, impresoras, gráficos o dispositivos externos.

Ejemplo:

print("Hola")

También puede mostrarse el contenido de variables:

nombre = "Ana"
print(nombre)

La salida es esencial porque permite comunicar al usuario lo que el programa ha hecho o cuál es el resultado de sus operaciones.

Entrada → proceso → salida

Una forma muy útil de entender los programas es mediante el modelo:

Entrada → Proceso → Salida

La entrada son los datos que recibe el programa. El proceso son las operaciones que realiza con esos datos. La salida es el resultado final.

Por ejemplo, para calcular el área de un rectángulo:

  • Entrada: base y altura
  • Proceso: multiplicar base por altura
  • Salida: el área calculada
base = float(input("Introduce la base: "))
altura = float(input("Introduce la altura: "))
area = base * altura
print("El área es:", area)

Expresiones y operadores básicos

Una expresión es una combinación de valores, variables y operadores que produce un resultado. Por ejemplo:

3 + 4
edad + 1

Los operadores aritméticos más habituales son:

  • + suma
  • - resta
  • * multiplicación
  • / división
  • // división entera
  • % resto
  • ** potencia

Ejemplo:

a = 10
b = 3

print(a + b)
print(a - b)
print(a * b)
print(a / b)
print(a // b)
print(a % b)
print(a ** b)

Conceptos clave

  • Programa: conjunto de instrucciones que el ordenador puede ejecutar.
  • Instrucción: orden concreta que el programa realiza.
  • Variable: espacio de memoria con nombre que almacena un dato.
  • Asignación: operación de guardar un valor en una variable.
  • Tipo de dato: clase de información almacenada, como entero, texto o booleano.
  • Entrada: datos que recibe el programa.
  • Salida: resultado que el programa muestra o produce.
  • Expresión: combinación de valores y operadores que produce un resultado.

Ejemplo práctico

Supongamos que queremos crear un programa que calcule la nota media de dos exámenes.

nota1 = float(input("Introduce la primera nota: "))
nota2 = float(input("Introduce la segunda nota: "))
media = (nota1 + nota2) / 2
print("La nota media es:", media)

Qué ocurre en este programa:

  • Se pide al usuario que introduzca dos notas.
  • Las entradas se convierten a números decimales.
  • Se calcula la media aritmética.
  • Se muestra el resultado en pantalla.

Qué conceptos aparecen aquí:

  • programa
  • variables
  • tipos de datos
  • entrada
  • salida
  • operadores

Aplicaciones en el mundo real

  • Calculadoras: reciben números, realizan operaciones y muestran resultados.
  • Formularios web: reciben datos del usuario y los almacenan o validan.
  • Videojuegos: guardan puntuaciones, posiciones y estados del juego en variables.
  • Apps móviles: procesan datos de entrada y generan salidas visuales o sonoras.
  • Robótica: los sistemas leen sensores, procesan información y actúan en consecuencia.

Aunque los programas reales sean mucho más complejos, todos dependen de estas ideas básicas.

Errores frecuentes de los estudiantes

  • Confundir variable con valor: una variable no es el dato en sí, sino el nombre que lo referencia.
  • Pensar que "=" significa igualdad matemática: en programación normalmente significa asignación.
  • No distinguir entre número y texto: 5 y "5" no son lo mismo.
  • Usar nombres poco claros: dificulta la comprensión del programa.
  • Olvidar convertir tipos: los datos introducidos por teclado suelen llegar como texto.
  • No entender el flujo entrada-proceso-salida: esto provoca muchos errores de razonamiento.

Resumen

  • Un programa es un conjunto de instrucciones que el ordenador puede ejecutar.
  • Las variables permiten almacenar datos usando nombres.
  • Los tipos de datos indican qué clase de información se guarda.
  • La entrada proporciona datos al programa.
  • La salida muestra o devuelve resultados.
  • Muchos programas siguen el modelo entrada → proceso → salida.
  • Estos fundamentos son la base de toda la programación posterior.

Preguntas tipo examen

  1. Define qué es un programa y explica en qué se diferencia de un algoritmo.
  2. Explica qué es una variable y por qué es importante en programación.
  3. Describe diferentes tipos de datos y pon un ejemplo de cada uno.
  4. Explica la diferencia entre entrada y salida en un programa.
  5. Analiza por qué es importante convertir correctamente los tipos de datos cuando un usuario introduce información.
  6. Explica el modelo entrada → proceso → salida con un ejemplo propio.
