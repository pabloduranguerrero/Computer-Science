Temario NM · A3.3

A3.3 Programación de bases de datos

Pregunta guía

¿Cómo se pueden consultar, añadir, modificar, eliminar y relacionar datos dentro de una base de datos utilizando SQL?

Introducción

Una base de datos bien diseñada no es útil por sí sola si no existe una forma de trabajar con la información que contiene. En la práctica, las aplicaciones informáticas necesitan acceder a los datos constantemente: consultar listas de usuarios, añadir nuevos registros, modificar información o eliminar datos que ya no son necesarios.

Para realizar estas operaciones se utiliza un lenguaje específico llamado SQL. SQL permite comunicarse con un sistema gestor de bases de datos para consultar y manipular la información almacenada. En este apartado estudiaremos los comandos básicos de SQL que permiten trabajar con bases de datos relacionales de forma eficaz.

Explicación detallada

SQL significa Structured Query Language. Es el lenguaje estándar utilizado para gestionar bases de datos relacionales. Gracias a SQL, los usuarios y las aplicaciones pueden interactuar con los datos sin necesidad de manipular directamente los archivos donde se almacenan.

Entre las operaciones más comunes que se realizan con SQL se encuentran consultar información, insertar nuevos registros, actualizar datos existentes, eliminar registros y combinar información de varias tablas relacionadas.

El comando SELECT se utiliza para consultar datos de una tabla. Permite obtener una o varias columnas, o incluso todos los datos almacenados. Por ejemplo:

SELECT Nombre
FROM Estudiantes;

Esta consulta devuelve la columna Nombre de la tabla Estudiantes.

Si queremos obtener todas las columnas de una tabla, usamos el símbolo *:

SELECT *
FROM Estudiantes;

El comando INSERT se utiliza para añadir nuevos registros a una tabla:

INSERT INTO Estudiantes (ID_estudiante, Nombre, Curso)
VALUES (3, 'Laura', 'DP1');

Con esta instrucción, la tabla añade un nuevo estudiante.

El comando UPDATE se utiliza para modificar registros existentes. Es muy importante usarlo junto con una condición para evitar cambiar todos los datos de la tabla:

UPDATE Estudiantes
SET Curso = 'DP2'
WHERE Nombre = 'Laura';

El comando DELETE se utiliza para eliminar registros:

DELETE FROM Estudiantes
WHERE Nombre = 'Laura';

De nuevo, la condición es esencial para indicar qué registros deben borrarse.

Además de trabajar con una sola tabla, SQL permite combinar datos de varias tablas. Para ello se utiliza INNER JOIN. Esta operación relaciona dos tablas usando un campo común y devuelve únicamente los registros que coinciden en ambas.

Por ejemplo, supongamos estas tablas:

Tabla Estudiantes

ID_estudiante Nombre
1 Ana
2 Carlos

Tabla Cursos

ID_estudiante Curso
1 Informática
2 Matemáticas

Consulta con INNER JOIN:

SELECT Estudiantes.Nombre, Cursos.Curso
FROM Estudiantes
INNER JOIN Cursos
ON Estudiantes.ID_estudiante = Cursos.ID_estudiante;

El resultado muestra el nombre de cada estudiante junto con su curso correspondiente. Esto demuestra cómo SQL puede combinar información distribuida en distintas tablas.

Conceptos clave

  • SQL: lenguaje estándar para gestionar bases de datos relacionales.
  • Consulta: instrucción que permite obtener o manipular datos.
  • SELECT: consulta para recuperar datos.
  • INSERT: instrucción para añadir nuevos registros.
  • UPDATE: instrucción para modificar datos existentes.
  • DELETE: instrucción para eliminar registros.
  • INNER JOIN: operación que combina datos de dos tablas relacionadas.
  • WHERE: cláusula que especifica una condición.

Ejemplo práctico

Imaginemos una base de datos de una tienda.

Tabla: Clientes

ID_cliente Nombre
1 Marta
2 Luis

Tabla: Pedidos

ID_pedido ID_cliente Producto
101 1 Libro
102 2 Calculadora

Consulta para mostrar el nombre del cliente y el producto pedido:

SELECT Clientes.Nombre, Pedidos.Producto
FROM Clientes
INNER JOIN Pedidos
ON Clientes.ID_cliente = Pedidos.ID_cliente;

Esta consulta permite ver claramente qué producto ha comprado cada cliente.

Aplicaciones en el mundo real

  • Plataformas educativas: consultan alumnado, tareas y calificaciones.
  • Tiendas online: añaden pedidos, actualizan stock y consultan productos.
  • Hospitales: consultan historiales y actualizan información de pacientes.
  • Bancos: registran operaciones y consultan movimientos de cuentas.
  • Empresas: generan informes a partir de bases de datos relacionales.

En todos estos casos, SQL permite acceder a la información de forma rápida y estructurada.

Errores frecuentes de los estudiantes

  • Olvidar la cláusula WHERE en UPDATE o DELETE: esto puede modificar o borrar todos los registros.
  • Confundir SELECT con INSERT: SELECT consulta datos; INSERT añade registros.
  • No identificar correctamente el campo común en INNER JOIN: esto impide relacionar bien las tablas.
  • Escribir mal los nombres de tablas o campos: la consulta deja de funcionar correctamente.
  • Pensar que SQL solo sirve para consultar: también sirve para insertar, actualizar, borrar y combinar datos.

Resumen

  • SQL es el lenguaje utilizado para interactuar con bases de datos relacionales.
  • SELECT permite consultar información.
  • INSERT permite añadir nuevos registros.
  • UPDATE permite modificar datos existentes.
  • DELETE permite eliminar registros.
  • INNER JOIN permite combinar datos de varias tablas relacionadas.
  • Las cláusulas y condiciones correctas son esenciales para trabajar con seguridad y precisión.

Preguntas tipo examen

  1. Define qué es SQL y explica por qué es importante en una base de datos relacional.
  2. Explica la diferencia entre SELECT, INSERT, UPDATE y DELETE.
  3. Describe qué hace INNER JOIN y por qué es útil.
  4. Analiza el riesgo de ejecutar una instrucción UPDATE o DELETE sin cláusula WHERE.
  5. Escribe un ejemplo de consulta SQL que combine datos de dos tablas relacionadas.
