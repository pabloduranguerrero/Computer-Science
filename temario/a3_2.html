Temario NM · A3.2

A3.2 Diseño de bases de datos

Pregunta guía

¿Cómo diseñar una base de datos que evite redundancias y errores?

Introducción

Diseñar bien una BD evita duplicidades y errores futuros.

Explicación detallada

El modelo relacional usa claves primarias para identificar registros y claves externas para relacionar tablas.

La normalización básica (1NF, 2NF, 3NF) reduce redundancia y mejora coherencia de datos.

Conceptos clave

  • Modelo relacional
  • Clave primaria
  • Clave externa
  • Normalización básica

Ejemplo práctico

Sistema de matrículas: Estudiantes, Asignaturas y Matrículas enlazadas por claves evita duplicar información.

Aplicaciones en el mundo real

ERP, sistemas escolares, comercio electrónico.

Errores frecuentes de los estudiantes

No definir PK; guardar listas dentro de una celda; relaciones mal planteadas.

Conexiones con otros temas del curso

A3.3 (consultas SQL), B1.1 (descomposición), A2 (servicios en red).

Resumen

  • Diseño correcto mejora calidad de datos.
  • PK/FK sostienen relaciones fiables.
  • Normalizar previene anomalías.
