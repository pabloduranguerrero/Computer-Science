Banco de preguntas (NM)

Práctica autocontenida para estudiar de forma autónoma: cada bloque incluye preguntas cortas, desarrollo, preguntas tipo examen y respuestas orientativas.

Cómo usar este banco

  1. Selecciona tema y subtema.
  2. Responde primero sin mirar la solución.
  3. Compara con la respuesta orientativa.
  4. Reescribe mejorando precisión técnica.

Bloque Prueba 1 (Área A + estudio de caso)

A1.2 Representación de datos y lógica

Pregunta corta: Convierte AC hexadecimal a binario.

Respuesta orientativa: AC = 1010 1100.

Desarrollo: Explica por qué las tablas de verdad son útiles para validar circuitos lógicos.

Modelo: Permiten comprobar sistemáticamente todas las combinaciones de entrada y garantizar la salida esperada.

A2.4 Seguridad de red

Pregunta corta: ¿Qué función cumple un firewall?

Respuesta orientativa: Filtra tráfico entrante/saliente según reglas.

Tipo examen: Evalúa la eficacia de un firewall en una red escolar.

Modelo: Útil como capa de defensa, pero insuficiente sin MFA, formación anti-phishing y actualización continua.

Bloque Prueba 2 (Área B)

B2.4 Algoritmos de programación

Pregunta corta: ¿Cuándo usar búsqueda binaria?

Respuesta orientativa: Cuando la lista está ordenada.

Desarrollo: Compara lineal vs binaria en eficiencia.

Modelo: Lineal O(n), binaria O(log n); en listas grandes ordenadas, binaria escala mejor.

B3.1 POO (una clase)

Pregunta corta: Diferencia clase y objeto.

Respuesta orientativa: Clase = plantilla; objeto = instancia concreta.

Tipo examen: Explica cómo encapsulación mejora fiabilidad del programa.

Modelo: Controla acceso al estado interno y reduce modificaciones inválidas.

Bloque Estudio de caso

Pregunta orientada a caso: Una institución educativa migra su plataforma a la nube y detecta problemas de rendimiento y seguridad. Analiza dos decisiones técnicas y evalúa ventajas/limitaciones.

Respuesta modelo (estructura): definir tecnología → aplicar al contexto → discutir beneficios y riesgos → concluir con recomendación priorizada.

Práctica autocontenida (mini bloque)

  1. Define NAT y da un ejemplo real.
  2. Escribe una consulta SQL con JOIN entre dos tablas.
  3. Explica una diferencia entre pila y cola.
  4. Evalúa una implicación ética de ML en educación.

Clave de corrección: precisión conceptual, aplicación, calidad de argumentación y claridad de redacción.
